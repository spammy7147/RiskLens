# Life Risk Simulator

개인의 생애 재무 리스크를 분석하고, 어떤 경제적 위험에 대비가 부족한지 수치화해주는 서비스입니다.

이 프로젝트는 특정 보험 상품을 추천하거나 판매하는 서비스가 아닙니다.

사용자의 소득, 지출, 자산, 부채, 가족 구성 정보를 바탕으로 사망, 질병, 소득중단 등의 리스크를 계산하고 사용자가 감당하기 어려운 경제적 위험을 파악할 수 있도록 돕는 것을 목표로 합니다.

# 문제 정의

대부분의 사람들은 보험이 필요한지 판단하기 어렵습니다.

사람들이 실제로 궁금한 것은 다음과 같습니다.

* 나는 보험이 필요한가?
* 필요하다면 왜 필요한가?
* 내가 가장 취약한 경제적 리스크는 무엇인가?
* 현재 자산으로 감당 가능한가?
* 부족한 대비 수준은 어느 정도인가?

하지만 대부분의 보험 서비스는 상품 판매 중심으로 접근합니다.

본 프로젝트는 상품 추천보다 개인의 재무 리스크를 정량적으로 분석하는 데 집중합니다.

---

# 프로젝트 목표

사용자의 재무 상태를 기반으로 다음을 제공한다.

* 경제적 리스크 식별
* 예상 손실 규모 계산
* 현재 대응 가능 수준 평가
* 부족한 대비 금액 산출
* 리스크 우선순위 제공

---

# 핵심 철학

> 보험을 추천하지 않는다.
>
> 위험을 계산한다.

> 상품을 팔지 않는다.
>
> 부족한 대비 수준을 보여준다.

> 정답을 제공하지 않는다.
>
> 의사결정에 필요한 숫자를 제공한다.

# 분석 대상 리스크

초기 버전에서는 다음 리스크를 분석한다.

## 1. 사망 리스크

가족이 상실하게 되는 미래 소득과 생활비를 계산한다.

### 예시 질문

* 내가 갑자기 사망하면 가족은 얼마나 버틸 수 있을까?
* 남겨진 부채는 어떻게 처리될까?
* 자녀 교육비는 충분할까?

---

## 2. 중대질병 리스크

질병 발생 시 발생 가능한 치료비와 소득 공백을 계산한다.

### 예시 질문

* 암에 걸린다면 경제적으로 얼마나 영향을 받을까?
* 몇 개월 동안 소득이 없어도 버틸 수 있을까?
* 현재 자산으로 감당 가능한 수준일까?

---

## 3. 소득중단 리스크

실직 또는 장기 휴직 시 현재 자산으로 얼마나 버틸 수 있는지 계산한다.

### 예시 질문

* 오늘 회사를 그만두게 된다면 몇 개월을 버틸 수 있을까?
* 현재 비상금은 충분한 수준일까?
* 가장 먼저 보완해야 할 부분은 무엇일까?

# 결과 예시

```text
당신의 가장 큰 리스크는 소득중단 리스크입니다.

예상 손실
84,000,000원

현재 대응 가능 금액
20,000,000원

부족 금액
64,000,000원
```

또는

```text
위험도 순위

1위 소득중단 리스크
2위 중대질병 리스크
3위 사망 리스크
```

사용자는 자신의 현재 상황에서 가장 취약한 리스크를 직관적으로 확인할 수 있어야 한다.

# MVP 범위

1차 목표는 아래 흐름을 완성하는 것이다.

```text
사용자 정보 입력
→ 리스크 분석
→ 결과 리포트 생성
→ 결과 저장
→ 분석 이력 조회
```

초기 버전에서는 다음 기능을 제외한다.

* 보험 상품 추천
* 보험사 API 연동
* 결제 기능
* 머신러닝
* 외부 공공데이터 연동
* 투자 상품 추천
* 자동 자산 연동

# 프로젝트가 해결하려는 문제

보험은 상품 중심으로 판매되지만 사용자는 위험 중심으로 생각한다.

사용자가 궁금한 것은 다음과 같다.

```text
어떤 보험을 가입해야 하지?
```

보다

```text
나는 어떤 위험에 노출되어 있지?
```

이다.

이 프로젝트는 상품 중심 접근이 아니라 리스크 중심 접근을 지향한다.

# 프로젝트 성공 기준

프로젝트의 성공은 보험 가입을 유도하는 것이 아니다.

사용자가 결과를 보고 다음 질문에 답할 수 있다면 성공이다.

* 내 가장 큰 경제적 위험은 무엇인가?
* 현재 자산으로 감당 가능한가?
* 부족한 대비 수준은 어느 정도인가?
* 어떤 위험부터 준비해야 하는가?

궁극적으로는 사용자가 자신의 재무적 취약성을 이해하고 더 나은 의사결정을 내릴 수 있도록 돕는 것을 목표로 한다.
