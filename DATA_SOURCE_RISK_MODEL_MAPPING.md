# 데이터 소스-분석지표-리스크 모델 매핑표

## 1. 문서 목적

이 문서는 Life Risk Simulator의 공공데이터 활용 방식, 분석 지표, 리스크 모델 반영 방식을 연결하기 위한 설계 문서이다.

프로젝트의 목적은 보험 상품 추천이 아니라, 공공 통계와 개인 재무 정보를 결합하여 사용자의 재무 취약성을 설명하는 것이다.

따라서 이 문서는 다음 질문에 답하는 것을 목표로 한다.

- 어떤 공공데이터를 사용할 것인가?
- 해당 데이터는 어떤 리스크 분석에 사용되는가?
- 개인 입력값과 어떻게 결합되는가?
- 어떤 분석 지표를 만들 것인가?
- 최종 리포트에서 어떻게 설명할 것인가?
- 어떤 한계와 주의사항이 있는가?

## 2. 분석 원칙

공공데이터는 개인의 미래를 단정적으로 예측하는 용도로 사용하지 않는다.

예를 들어 다음 표현은 피한다.

```text
당신은 암에 걸릴 확률이 높습니다.
당신은 사망할 가능성이 높습니다.
당신은 실직할 가능성이 높습니다.
```

대신 다음처럼 사용한다.

```text
공공 통계상 동일 연령대/성별/가구유형의 위험 맥락을 참고했을 때,
이 사용자는 위험 발생 시 재무 충격, 유동성 부족, 회복 기간 측면에서 취약합니다.
```

핵심 원칙:

- 공공 발생률은 최종 점수의 중심이 아니라 보정값이다.
- 최종 판단은 발생 가능성보다 재무 충격과 대응 가능성에 둔다.
- 평균값보다 중위값, 분위값, 연령대별 기준값을 우선 사용한다.
- 보험 상품명이 아니라 리스크 대응 가능 금액으로 해석한다.

## 3. 전체 분석 구조

```text
Public Data
  -> Public Baseline Metrics
  -> Personal Finance Metrics
  -> Scenario Simulation
  -> Risk Sub Scores
  -> Final Risk Score
  -> Explainable Report
```

레이어별 역할:

| Layer | 역할 | 예시 |
| --- | --- | --- |
| Public Data | 공공 통계 원천 데이터 | 사망률, 암 발생률, 실업률, 가계소득 |
| Public Baseline Metrics | 비교 기준값 생성 | 동일 연령대 중위소득, 연령별 암 발생률 |
| Personal Finance Metrics | 개인 재무 구조 계산 | 유동자산, 필수지출, 월 저축가능액 |
| Scenario Simulation | 리스크 발생 시 손실 계산 | 사망, 중대질병, 소득중단 |
| Risk Sub Scores | 하위 점수 산출 | 재무충격, 부족률, 유동성, 회복난이도 |
| Final Risk Score | 최종 점수와 순위 | 0~100점, 낮음/보통/높음/매우 높음 |
| Explainable Report | 결과 해석 | 왜 이 리스크가 취약한지 설명 |

## 5. 데이터 소스별 매핑

### 5.1 KOSIS 사망/인구 통계

| 항목 | 내용 |
| --- | --- |
| 데이터 후보 | 연령별 사망률, 성별 사망률, 사망원인별 사망률, 기대수명 |
| 개인 입력값 | 나이, 성별, 결혼 여부, 자녀 수, 부양가족 수, 배우자 소득 |
| 생성 지표 | Public Mortality Context Score, Age Mortality Band, Death Cause Context |
| 사용 리스크 | 사망 리스크 |
| 모델 반영 | Public Risk Context Score의 일부 |
| 리포트 설명 | 동일 연령대/성별의 사망 통계 맥락을 참고하여 사망 리스크의 공공 통계 보정값을 계산 |
| 주의사항 | 개인의 사망 가능성을 예측하지 않는다. 재무 충격 계산의 보조 맥락으로만 사용한다. |

사망 리스크에서의 역할:

```text
사망 리스크 점수 =
재무 충격 점수
+ 준비 부족률 점수
+ 가족 의존도 점수
+ 회복 난이도 점수
+ 공공 사망률 맥락 점수
```

### 5.2 KOSIS 가계금융복지조사/가계동향조사

| 항목 | 내용 |
| --- | --- |
| 데이터 후보 | 연령대별 소득, 가구유형별 지출, 자산, 부채, 순자산, 분위별 분포 |
| 개인 입력값 | 월 소득, 배우자 소득, 월 지출, 자산, 부채, 비상금, 가구원 수 |
| 생성 지표 | Income Percentile, Asset Percentile, Debt Burden Index, Expense Ratio |
| 사용 리스크 | 사망, 중대질병, 소득중단 전체 |
| 모델 반영 | Financial Impact Score, Liquidity Stress Score, Recovery Difficulty Score |
| 리포트 설명 | 동일 연령대/가구유형 기준으로 사용자의 소득, 자산, 부채, 지출 수준을 비교 |
| 주의사항 | 평균값은 고소득/고자산 가구에 의해 왜곡될 수 있으므로 중위값과 분위값을 우선 사용한다. |

핵심 파생 지표:

```text
소득 대비 지출률 = 월 필수지출 / 월 세후소득
부채상환부담률 = 월 부채상환액 / 월 세후소득
유동자산 비율 = 유동자산 / 총자산
순자산 = 총자산 - 총부채
```

### 5.3 국가암지식정보센터 암 통계

| 항목 | 내용 |
| --- | --- |
| 데이터 후보 | 암 발생률, 암종별 발생 현황, 연령군별 암발생률, 암 생존율, 5년 상대생존율, 암 유병률, 암 사망률 |
| 개인 입력값 | 나이, 성별, 월 소득, 월 필수지출, 질병 보장금액, 유동자산 |
| 생성 지표 | Cancer Incidence Context Score, Cancer Survival Context, Illness Duration Assumption |
| 사용 리스크 | 중대질병 리스크 |
| 모델 반영 | Public Risk Context Score, Recovery Difficulty Score 보조 |
| 리포트 설명 | 연령군별 암 발생률과 생존율을 참고하여 중대질병 리스크의 통계적 맥락을 반영 |
| 주의사항 | 암 발생률은 개인 질병 예측이 아니다. 초기 버전에서는 암을 중대질병 대표 시나리오로 사용한다. |

중대질병 리스크에서의 역할:

```text
중대질병 필요금액 =
예상 치료비
+ 치료기간 중 필수생활비
+ 치료기간 중 부채상환액
+ 간병/추가생활비
- 질병 관련 대응 가능 자산
- 질병 관련 보장금액
```

### 5.4 건강보험심사평가원/국민건강보험공단 의료비 통계

| 항목 | 내용 |
| --- | --- |
| 데이터 후보 | 질병군별 진료비, 입원 진료비, 외래 진료비, 주요 수술 통계, 연령대별 의료 이용량 |
| 개인 입력값 | 나이, 성별, 유동자산, 질병 보장금액, 실손 여부, 월 소득 |
| 생성 지표 | Medical Cost Baseline, Treatment Cost Scenario, Out-of-Pocket Risk Proxy |
| 사용 리스크 | 중대질병 리스크 |
| 모델 반영 | Financial Impact Score, Coverage Gap Score |
| 리포트 설명 | 공공 의료비 통계를 참고해 치료비 시나리오의 기준값을 설정 |
| 주의사항 | 공공 진료비 통계는 총진료비 또는 급여 중심일 수 있다. 실제 본인부담금, 비급여, 간병비는 별도 가정이 필요하다. |

초기 모델에서는 다음처럼 단순화한다.

```text
중대질병 치료비 기준값 =
공공 의료비 기준값
+ 비급여/간병비 보정값
```

### 5.5 KOSIS/고용노동부 고용 통계

| 항목 | 내용 |
| --- | --- |
| 데이터 후보 | 연령대별 실업률, 고용률, 산업별 고용 현황, 고용형태별 통계, 실업급여 관련 통계 |
| 개인 입력값 | 나이, 성별, 직업, 산업군, 고용형태, 근속연수, 월 소득, 비상금, 유동자산 |
| 생성 지표 | Employment Stability Context, Income Interruption Context Score, Reemployment Assumption |
| 사용 리스크 | 소득중단 리스크 |
| 모델 반영 | Public Risk Context Score, Recovery Difficulty Score |
| 리포트 설명 | 동일 연령대/고용형태의 고용 통계를 참고하여 소득중단 리스크 맥락을 반영 |
| 주의사항 | 실업률은 개인 실직 확률이 아니다. 소득중단 시나리오의 기간과 점수 보정에만 사용한다. |

소득중단 리스크에서의 역할:

```text
소득중단 필요금액 =
목표 생존 개월 수 * (월 필수지출 + 월 부채상환액)
- 소득중단 중 예상 유입액
- 즉시 사용 가능한 유동자산
```

## 6. 리스크별 통합 매핑표

### 6.1 사망 리스크

| 구분 | 내용 |
| --- | --- |
| 핵심 질문 | 본인의 소득이 사라졌을 때 남은 가족이 필요한 돈은 얼마인가? |
| 개인 입력값 | 나이, 성별, 배우자 여부, 배우자 소득, 자녀 수, 자녀 나이, 부양가족 수, 월 소득, 월 필수지출, 부채, 사망 보장금액 |
| 공공데이터 | KOSIS 연령/성별 사망률, 사망원인, 기대수명, 가계지출 기준 |
| 주요 계산값 | 가족 생활비 필요액, 자녀 교육비, 부채 정리 금액, 장례/정리 비용 |
| 주요 지표 | Expected Required Amount, Coverage Gap, Dependency Impact Score, Public Mortality Context |
| 점수 반영 | 재무충격 30%, 준비부족 25%, 가족영향 20%, 회복난이도 15%, 공공위험맥락 10% |
| 리포트 표현 | 사망 시나리오에서 부양가족 생활비와 잔여 부채를 감당하기 위한 준비금이 부족한지 분석 |

초기 계산식 후보:

```text
사망 필요금액 =
가족 월 필수생활비 * 필요보장개월
+ 자녀 교육비
+ 상환 필요 부채
+ 장례/정리 비용
- 배우자 소득 반영액
- 사망 관련 보장금액
- 사망 시 활용 가능한 유동자산
```

### 6.2 중대질병 리스크

| 구분 | 내용 |
| --- | --- |
| 핵심 질문 | 질병으로 인해 치료비와 소득 공백이 발생하면 얼마가 부족한가? |
| 개인 입력값 | 나이, 성별, 월 소득, 월 필수지출, 월 부채상환액, 유동자산, 질병 보장금액, 실손 여부 |
| 공공데이터 | 국가암지식정보센터 암 발생률/생존율/유병률, 심평원/건보 의료비 통계 |
| 주요 계산값 | 치료비, 간병비, 소득공백 기간 생활비, 부채상환액, 보장금액 |
| 주요 지표 | Medical Cost Baseline, Illness Income Gap, Coverage Gap, Recovery Months |
| 점수 반영 | 재무충격 30%, 준비부족 25%, 유동성압박 20%, 회복난이도 15%, 공공위험맥락 10% |
| 리포트 표현 | 암 중심 중대질병 시나리오에서 치료비와 소득공백을 감당할 수 있는지 분석 |

초기 계산식 후보:

```text
중대질병 필요금액 =
공공 의료비 기준 치료비
+ 비급여/간병비 보정
+ 치료기간개월 * (월 필수지출 + 월 부채상환액)
- 치료기간 중 유지 가능 소득
- 질병 관련 보장금액
- 질병 시 활용 가능한 유동자산
```

### 6.3 소득중단 리스크

| 구분 | 내용 |
| --- | --- |
| 핵심 질문 | 소득이 끊겼을 때 몇 개월 버틸 수 있고, 목표 기간 대비 얼마가 부족한가? |
| 개인 입력값 | 나이, 성별, 직업, 산업군, 고용형태, 근속연수, 월 소득, 배우자 소득, 월 필수지출, 월 부채상환액, 비상금, 유동자산 |
| 공공데이터 | KOSIS 실업률/고용률, 고용노동부 고용통계, 실업급여 관련 통계 |
| 주요 계산값 | 생존 가능 개월 수, 목표 생존 개월 수, 부족 개월 수, 부족 금액 |
| 주요 지표 | Liquidity Survival Months, Income Interruption Gap, Debt Payment Stress, Recovery Months |
| 점수 반영 | 재무충격 25%, 준비부족 20%, 유동성압박 30%, 회복난이도 15%, 공공위험맥락 10% |
| 리포트 표현 | 소득중단 시 현재 비상금과 유동자산으로 버틸 수 있는 기간을 분석 |

초기 계산식 후보:

```text
월 생존비용 = 월 필수지출 + 월 부채상환액

현재 생존 가능 개월 수 =
즉시 사용 가능한 유동자산 / 월 생존비용

소득중단 부족금액 =
목표 생존 개월 수 * 월 생존비용
- 소득중단 중 예상 유입액
- 즉시 사용 가능한 유동자산
```

## 7. 분석 지표 상세 정의

| 지표 | 계산 방식 | 사용 리스크 | 해석 |
| --- | --- | --- | --- |
| Financial Impact Ratio | 예상 필요금액 / 연 세후소득 | 전체 | 손실이 연소득 대비 얼마나 큰지 |
| Coverage Gap | 예상 필요금액 - 대응 가능 금액 | 전체 | 부족한 준비금 |
| Coverage Gap Ratio | 부족금액 / 예상 필요금액 | 전체 | 필요금액 중 준비되지 않은 비율 |
| Coverage Ratio | 대응 가능 금액 / 예상 필요금액 | 전체 | 현재 준비 수준 |
| Liquidity Survival Months | 유동자산 / 월 생존비용 | 소득중단, 질병 | 현금성 자산으로 버틸 수 있는 기간 |
| Recovery Months | 부족금액 / 월 저축가능액 | 전체 | 부족금액 회복에 필요한 기간 |
| Debt Payment Stress | 월 부채상환액 / 월 세후소득 | 전체 | 소득 대비 부채상환 부담 |
| Income Dependency Ratio | 본인 소득 / 가구 총소득 | 사망, 질병 | 가구가 본인 소득에 의존하는 정도 |
| Essential Expense Ratio | 월 필수지출 / 월 총지출 | 전체 | 줄이기 어려운 지출의 비중 |
| Liquid Asset Ratio | 유동자산 / 총자산 | 전체 | 자산의 현금화 가능성 |
| Public Risk Context Score | 공공통계 기준 상대 위험도 | 전체 | 발생 가능성 맥락 보정 |

## 8. 최종 점수 구조

최종 점수는 0~100점으로 표현한다.

```text
Final Risk Score =
Financial Impact Score
* financialImpactWeight
+ Coverage Gap Score
* coverageGapWeight
+ Liquidity Stress Score
* liquidityStressWeight
+ Recovery Difficulty Score
* recoveryDifficultyWeight
+ Public Risk Context Score
* publicRiskContextWeight
```

초기 공통 가중치:

| 하위 점수 | 가중치 | 의미 |
| --- | --- | --- |
| Financial Impact Score | 30% | 위험 발생 시 손실 규모 |
| Coverage Gap Score | 25% | 준비 부족률 |
| Liquidity Stress Score | 20% | 단기 현금 대응력 |
| Recovery Difficulty Score | 15% | 회복 소요 기간 |
| Public Risk Context Score | 10% | 공공통계 기반 위험 맥락 |

리스크별로 가중치를 조정할 수 있다.

| 리스크 | 재무충격 | 준비부족 | 유동성 | 회복난이도 | 공공위험맥락 |
| --- | ---: | ---: | ---: | ---: | ---: |
| 사망 | 30% | 25% | 10% | 15% | 20% |
| 중대질병 | 30% | 25% | 20% | 15% | 10% |
| 소득중단 | 25% | 20% | 30% | 15% | 10% |

주의:

- 사망 리스크는 가족 의존도와 공공 사망률 맥락을 조금 더 반영한다.
- 소득중단 리스크는 유동성 압박을 가장 크게 반영한다.
- 중대질병 리스크는 치료비와 소득공백을 함께 반영한다.

## 9. 리포트 설명 매핑

| 분석 결과 | 사용자에게 보여줄 설명 |
| --- | --- |
| 부족금액 높음 | 해당 시나리오에서 현재 확인된 대응 가능 금액보다 필요한 금액이 큽니다. |
| 대응률 낮음 | 예상 필요금액 중 준비된 금액의 비율이 낮습니다. |
| 생존 가능 개월 수 낮음 | 소득이 중단될 경우 현재 유동자산으로 버틸 수 있는 기간이 짧습니다. |
| 회복기간 김 | 부족금액을 현재 저축 가능액으로 회복하는 데 오랜 시간이 필요합니다. |
| 부채상환부담 높음 | 월 소득 중 부채상환에 사용되는 비율이 높아 현금흐름 압박이 큽니다. |
| 소득의존도 높음 | 가구 소득이 특정 소득원에 집중되어 있어 사망/질병/소득중단 시 충격이 큽니다. |
| 공공위험맥락 높음 | 동일 연령대/성별/고용형태의 공공 통계상 해당 리스크를 참고 보정했습니다. |

피해야 할 표현:

```text
보험 가입이 필요합니다.
암보험이 부족합니다.
정기보험을 추천합니다.
당신은 위험한 사람입니다.
```

사용할 표현:

```text
이 시나리오에서는 현재 대응 가능 금액이 부족합니다.
소득중단 시나리오에서 유동성 압박이 크게 나타납니다.
중대질병 시나리오에서 치료비보다 소득공백의 영향이 더 크게 나타납니다.
사망 시나리오에서 부양가족의 생활비 의존도가 주요 취약 요인입니다.
```

## 10. 데이터 수집 우선순위

초기에는 모든 데이터를 API로 자동 연동하지 않는다.

포트폴리오 안정성과 재현성을 위해 다음 순서를 권장한다.

### Phase 1. 수동 수집 기반 정적 데이터셋

- KOSIS, 국가암지식정보센터, 공공데이터포털에서 필요한 통계를 CSV/JSON으로 정리
- 프로젝트 내부에 샘플 기준 데이터로 보관
- 계산 로직과 리포트 생성에 집중

### Phase 2. API 연동

- KOSIS OpenAPI 후보 검토
- 공공데이터포털 REST API 후보 검토
- API 응답을 내부 기준 데이터 형태로 변환

### Phase 3. 데이터 버전 관리

- 기준 통계 연도 명시
- 데이터 출처 URL 명시
- 수집일 명시
- 통계 변경 시 재계산 가능하도록 버전 필드 관리

## 12. 질병별 의료비 지표 설계

중대질병 리스크에서 가장 중요한 공공 지표는 단순 발생률이 아니라, **질병 발생 시 실제로 어느 정도의 의료비와 치료 기간이 발생하는가**이다.

따라서 중대질병 모델은 다음 데이터를 중심으로 설계한다.

```text
질병 발생률
+ 질병별 평균 진료비
+ 치료/입원 기간
+ 소득공백 기간
+ 비급여/간병비 보정
= 중대질병 재무 충격
```

### 12.1 핵심 데이터 출처

| 출처 | 확인할 메뉴/데이터 | 활용 목적 |
| --- | --- | --- |
| 건강보험심사평가원 보건의료빅데이터개방시스템 | 국민관심질병통계 | 주요 질병의 환자 수, 진료비, 진료 현황 확인 |
| 건강보험심사평가원 보건의료빅데이터개방시스템 | 다빈도질병 통계 | 자주 발생하는 질병과 진료비 규모 확인 |
| 건강보험심사평가원 보건의료빅데이터개방시스템 | 질병 소분류(3단 상병) 통계 | 질병군 단위의 진료비 기준값 생성 |
| 건강보험심사평가원 보건의료빅데이터개방시스템 | 질병 세분류(4단 상병) 통계 | 더 구체적인 질병별 비용 분석 |
| 건강보험심사평가원 보건의료빅데이터개방시스템 | 진료행위(검사/수술 등) 통계 | 수술, 검사, 처치 비용 시나리오 보강 |
| 건강보험심사평가원 보건의료빅데이터개방시스템 | 질병별 의약품 통계 | 장기 약물치료 비용 참고 |
| 국가암지식정보센터 | 암 발생률, 암종별 발생, 연령군별 암발생률 | 암 발생 맥락 보정 |
| 국가암지식정보센터 | 암 생존율, 5년 상대생존율, 유병률 | 장기 치료와 회복 부담 설명 |

### 12.2 수집해야 할 원천 지표

| 원천 지표 | 의미 | 모델 활용 |
| --- | --- | --- |
| 질병별 환자 수 | 해당 질병으로 진료받은 인원 | 발생 빈도와 대중성 확인 |
| 질병별 총진료비 | 해당 질병으로 발생한 전체 진료비 | 질병의 사회적 비용 규모 |
| 1인당 평균 진료비 | 총진료비 / 환자 수 | 개인 치료비 시나리오 기준값 |
| 입원 진료비 | 입원 중심 치료비 | 중증도 보정 |
| 외래 진료비 | 외래 중심 치료비 | 장기 관리 비용 보정 |
| 입원일수 | 입원 치료 기간 | 소득공백 기간 추정 |
| 내원일수 | 병원 이용 빈도 | 치료 지속 기간과 생산성 손실 추정 |
| 연령별 진료비 | 연령대에 따른 비용 차이 | 사용자 나이 기반 개인화 |
| 성별 진료비 | 성별에 따른 비용 차이 | 사용자 성별 기반 개인화 |
| 검사/수술 행위 비용 | 주요 치료 행위 비용 | 세부 시나리오 보강 |

핵심 파생 지표:

```text
질병별 1인당 평균 진료비 =
질병별 총진료비 / 질병별 환자 수
```

입원 기반 중증도 지표:

```text
입원비 비중 =
입원 진료비 / 전체 진료비
```

치료기간 추정 지표:

```text
평균 입원일수 =
총 입원일수 / 입원 환자 수
```

### 12.3 중대질병 비용 모델

공공 의료비 통계는 중대질병 필요금액의 출발점으로 사용한다.

```text
중대질병 필요금액 =
질병별 1인당 평균 진료비
+ 비급여/간병비 보정
+ 치료기간 중 필수생활비
+ 치료기간 중 부채상환액
- 치료기간 중 유지 가능 소득
- 질병 관련 보장금액
- 질병 시 활용 가능한 유동자산
```

이때 질병별 1인당 평균 진료비만으로는 부족하다.

재무 리스크 관점에서는 다음 항목이 함께 중요하다.

- 치료비
- 비급여 가능 비용
- 간병비
- 입원 기간
- 통원 치료 기간
- 소득공백 기간
- 보호자 돌봄으로 인한 가구 소득 감소
- 치료 후 회복 기간
- 기존 부채상환 지속 부담

### 12.4 초기 질병군 후보

초기 MVP에서는 질병 범위를 너무 넓히지 않는다.

다음 3개 질병군을 우선 후보로 둔다.

| 질병군 | 선택 이유 | 주요 데이터 |
| --- | --- | --- |
| 암 | 발생률, 생존율, 유병률 등 공공 통계가 잘 정리되어 있고 중대질병 대표성이 높음 | 암 발생률, 암종별 발생, 5년 생존율, 암 진료비 |
| 심혈관질환 | 갑작스러운 치료비와 소득공백, 수술/입원 가능성이 큼 | 질병별 진료비, 입원일수, 수술/검사 통계 |
| 뇌혈관질환 | 치료 이후 장기 회복, 간병, 소득상실 가능성이 큼 | 질병별 진료비, 입원일수, 재활/장기치료 통계 |

확장 후보:

- 희귀/중증난치질환
- 만성질환 장기 관리 비용
- 장기입원 질환
- 주요 수술 중심 질환
- 정신건강 관련 장기 소득손실 리스크

### 12.5 암 모델의 데이터 결합 방식

암 리스크는 단일 출처가 아니라 두 축을 결합한다.

```text
국가암지식정보센터:
암 발생률, 암종별 발생 현황, 연령군별 발생률, 생존율, 유병률

건강보험심사평가원/국민건강보험:
암 관련 진료비, 입원/외래 진료비, 수술/검사 통계

사용자 재무 입력:
월 소득, 월 필수지출, 유동자산, 부채상환액, 질병 보장금액
```

분석 문장 예시:

```text
암 발생률은 공공 통계 기반 위험 맥락으로만 반영하고,
실제 리스크 점수는 암 발생 시 예상 치료비, 소득공백, 유동성 부족, 회복기간을 중심으로 계산합니다.
```

### 12.6 의료비 통계 사용 시 주의사항

건강보험 진료비 통계는 사용자가 실제로 부담하는 모든 비용과 같지 않다.

주의해야 할 차이:

| 구분 | 설명 |
| --- | --- |
| 총진료비 | 공단부담금과 본인부담금이 함께 포함될 수 있음 |
| 급여 진료비 | 건강보험 적용 항목 중심 |
| 비급여 비용 | 통계에 충분히 반영되지 않을 수 있음 |
| 간병비 | 의료비 통계에 직접 반영되지 않을 수 있음 |
| 생활비 | 치료기간 중 계속 발생하지만 진료비에는 포함되지 않음 |
| 소득공백 | 의료비가 아니라 재무 손실로 별도 계산해야 함 |
| 보호자 비용 | 보호자 돌봄, 교통비, 숙박비 등은 별도 보정 필요 |

따라서 모델에서는 의료비를 다음 3단계로 분리한다.

```text
1. 공공통계 기반 의료비
   질병별 평균 진료비

2. 개인 직접 부담 보정
   본인부담률, 비급여 보정률, 간병비 보정

3. 생활/소득 충격
   치료기간 중 소득공백, 필수생활비, 부채상환액
```

### 12.7 데이터셋 설계 후보

초기 정적 데이터셋은 다음 형태를 권장한다.

```text
medical_cost_baseline
```

| 필드 | 설명 |
| --- | --- |
| disease_group | 암, 심혈관질환, 뇌혈관질환 등 상위 질병군 |
| disease_code | KCD 또는 통계에서 사용하는 질병 코드 |
| disease_name | 질병명 |
| age_band | 연령대 |
| gender | 성별 |
| patient_count | 환자 수 |
| total_medical_cost | 총진료비 |
| avg_cost_per_patient | 1인당 평균 진료비 |
| inpatient_cost | 입원 진료비 |
| outpatient_cost | 외래 진료비 |
| avg_hospital_days | 평균 입원일수 |
| avg_visit_days | 평균 내원일수 |
| source_name | 데이터 출처 |
| source_year | 기준 연도 |
| collected_at | 수집일 |

중대질병 시나리오용 보정 데이터셋:

```text
illness_scenario_assumption
```

| 필드 | 설명 |
| --- | --- |
| disease_group | 질병군 |
| severity_level | mild, moderate, severe |
| income_loss_months | 소득공백 개월 수 |
| recovery_months | 회복 기간 |
| non_covered_cost_multiplier | 비급여/추가비용 보정 계수 |
| caregiver_cost_monthly | 월 간병비 가정 |
| description | 가정 설명 |

### 12.8 리포트 표현 방식

질병별 의료비 지표는 사용자에게 다음처럼 설명한다.

```text
중대질병 시나리오는 공공 의료비 통계의 질병별 평균 진료비를 기준으로 계산했습니다.
다만 실제 부담액은 비급여, 간병비, 치료 방식, 소득공백 여부에 따라 달라질 수 있어
본 분석에서는 의료비와 생활/소득 충격을 분리해 추정했습니다.
```

피해야 할 표현:

```text
이 병에 걸리면 정확히 이 금액이 듭니다.
당신은 이 질병에 걸릴 가능성이 높습니다.
이 질병에 대비하려면 특정 보험이 필요합니다.
```

사용할 표현:

```text
공공 통계상 해당 질병군의 평균 진료비를 기준으로 치료비 시나리오를 구성했습니다.
사용자의 유동자산과 질병 관련 대응금액을 반영했을 때 부족 금액은 다음과 같습니다.
이 결과는 보험 추천이 아니라 중대질병 발생 시 재무 충격을 추정한 것입니다.
```

## 13. 테스트 설계로 이어지는 포인트

이 매핑표는 이후 테스트 설계의 기준이 된다.

테스트해야 할 핵심 케이스:

- 유동자산이 0원인 경우
- 월 저축 가능액이 0 이하인 경우
- 부양가족이 없는 경우의 사망 리스크
- 배우자 소득이 높은 경우의 사망 리스크 완화
- 총자산은 크지만 유동자산이 낮은 경우
- 부채상환액이 월 소득 대비 과도한 경우
- 질병 보장금액은 있으나 소득공백 대응금이 부족한 경우
- 공공 의료비 기준값은 낮지만 입원/회복기간이 길어 소득공백 손실이 큰 경우
- 총진료비는 크지만 1인당 평균 진료비는 낮은 질병군
- 환자 수는 적지만 1인당 평균 진료비가 높은 질병군
- 비급여/간병비 보정 여부에 따라 부족금액이 크게 달라지는 경우
- 소득중단 목표기간보다 생존 가능 개월 수가 짧은 경우
- 공공 통계값이 없거나 매칭되지 않는 연령대/직업군

## 14. 현재 결정 사항

현재까지의 설계 결론은 다음과 같다.

```text
프로젝트 유형:
공공통계 기반 정밀 재무 리스크 분석기

분석 방식:
시나리오 기반 재무 충격 계산 + 공공통계 위험 맥락 보정

핵심 리스크:
사망, 중대질병, 소득중단

공공데이터 역할:
기준선, 시나리오 근거, 점수 보정

점수 철학:
재무 충격과 준비 부족 중심
공공 발생률은 보조 지표

구현 우선순위:
정적 기준 데이터셋 -> 계산 로직 -> 리포트 -> API 연동
```
