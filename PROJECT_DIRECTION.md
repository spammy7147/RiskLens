# Life Risk Simulator Project Direction

## 1. Project Positioning

Life Risk Simulator는 보험 상품 추천 서비스가 아니라, 개인의 생애 재무 리스크를 분석하는 서비스이다.

이 프로젝트의 핵심 관점은 다음과 같다.

- 보험을 추천하지 않는다.
- 특정 상품을 판매하지 않는다.
- 사용자의 재무 구조가 어떤 위험에 취약한지 계산한다.
- 위험 발생 시 예상 재무 충격과 부족 금액을 보여준다.
- 공공 통계를 근거로 분석 방법과 해석 과정을 설명한다.

따라서 프로젝트의 목표는 단순 계산기가 아니라, **공공데이터 기반 정밀 재무 리스크 분석기**를 만드는 것이다.

## 2. Development Goal

이 프로젝트는 빠르게 기능을 완성하는 것보다, 다음 역량을 보여주는 것을 중요하게 둔다.

- 도메인 문제 정의
- 공공데이터 기반 분석 설계
- 재무 리스크 계산식 설계
- 설명 가능한 점수 모델 설계
- 백엔드 도메인 구조 설계
- 테스트 가능한 계산 로직 설계
- 분석 결과를 사용자에게 이해 가능한 리포트로 표현하는 능력

즉, 프로젝트의 목적은 단순한 보험 계산기가 아니라, **분석 방법과 그 이유를 논리적으로 설명할 수 있는 포트폴리오 프로젝트**이다.

## 3. Core Analysis Direction

초기 방향은 다음과 같이 정리한다.

```text
위험 발생 가능성 예측 중심이 아니라,
위험 발생 시 재무 충격과 부족 금액을 계산하는 시나리오 분석을 중심으로 한다.
```

다만 공공 통계 기반의 발생률, 사망률, 질병 통계, 고용 통계 등을 보조 지표로 사용하여 최종 점수에 반영한다.

정리하면 다음 구조가 된다.

```text
시나리오 기반 손실 계산
+ 개인 재무 대응력 분석
+ 공공 통계 기반 위험 맥락
= 설명 가능한 재무 리스크 점수
```

## 4. Public Data Strategy

공공데이터는 사용자의 미래를 단정적으로 예측하는 용도가 아니다.

예를 들어 다음과 같은 표현은 피한다.

```text
당신은 암에 걸릴 확률이 높습니다.
당신은 사망 리스크가 높으므로 보험이 필요합니다.
```

대신 다음과 같이 해석한다.

```text
동일 연령대의 공공 통계상 질병/사망/고용 리스크를 참고했을 때,
이 사용자는 발생 가능성보다 발생 시 재무 충격과 회복 기간 측면에서 취약합니다.
```

공공데이터는 다음 3가지 용도로 사용한다.

1. 기준선
   - 동일 연령대, 성별, 가구유형, 소득구간과 비교한다.

2. 시나리오 근거
   - 사망, 중대질병, 소득중단 시나리오의 평균값이나 기준값을 설정한다.

3. 점수 보정
   - 공공 통계상 상대적으로 빈번하거나 중요한 위험을 최종 점수에 일부 반영한다.

```text
비급여, 병원 이용 형태, 실손 여부, 보장 범위에 따라 실제 부담액은 달라진다.
```

## 6. MVP Risk Scope

초기 MVP에서는 리스크 종류를 많이 늘리지 않는다.

대신 핵심 리스크 3개를 깊게 분석한다.

```text
1. 사망 리스크
2. 중대질병 리스크
3. 소득중단 리스크
```

이후 확장 후보:

- 장수 리스크
- 은퇴 리스크
- 주거 리스크
- 육아 리스크
- 노후 의료비 리스크
- 산업재해 리스크

## 7. Risk Analysis Model

전체 분석 모델은 4개 레이어로 나눈다.

```text
1. Public Baseline Layer
   한국 공공통계 기반 기준값

2. Personal Finance Layer
   사용자의 소득, 지출, 자산, 부채, 가족구성

3. Scenario Simulation Layer
   사망, 중대질병, 소득중단 발생 시 재무 충격 계산

4. Risk Scoring Layer
   손실규모, 부족률, 유동성, 회복기간, 공공 발생률을 점수화
```

이 구조의 장점:

- 사용자 입력값만 계산하지 않는다.
- 공공 통계로 기준선을 만든다.
- 개인의 재무 구조를 기준선과 비교한다.
- 위험 발생 시나리오에서 부족금액과 회복기간을 계산한다.
- 최종 결과의 근거를 설명할 수 있다.

## 8. Core Metrics

각 리스크 분석 결과에는 다음 지표를 포함한다.

### 8.1 Expected Required Amount

위험 발생 시 필요한 총 금액이다.

예:

- 사망 후 가족 생활비
- 질병 치료비
- 소득중단 기간의 필수지출
- 부채상환액
- 간병비
- 자녀 교육비

### 8.2 Available Protection Amount

현재 대응 가능한 금액이다.

단, 총자산 전체를 대응 가능 금액으로 보지 않는다.

자산은 유동성에 따라 분리한다.

- 현금성 자산
- 단기 유동자산
- 투자자산
- 부동산/전세보증금
- 퇴직연금
- 보험/보장성 자산

### 8.3 Coverage Gap

부족 금액이다.

```text
부족 금액 = 예상 필요금액 - 현재 대응 가능 금액
```

### 8.4 Coverage Ratio

대응률이다.

```text
대응률 = 현재 대응 가능 금액 / 예상 필요금액
```

### 8.5 Liquidity Survival Months

소득이 끊겼을 때 몇 개월 버틸 수 있는지 나타내는 지표이다.

```text
생존 가능 개월 수 = 유동자산 / 월 필수지출
```

부채상환액을 포함할 경우:

```text
생존 가능 개월 수 = 유동자산 / (월 필수지출 + 월 부채상환액)
```

### 8.6 Recovery Months

부족금액을 현재 저축 능력으로 회복하는 데 필요한 기간이다.

```text
월 저축 가능액 = 월 소득 - 월 지출 - 월 부채상환액
회복 소요 개월 수 = 부족 금액 / 월 저축 가능액
```

월 저축 가능액이 0 이하라면 별도 상태로 처리한다.

```text
현금흐름 적자 상태
회복 기간 산정 불가
```

## 9. Risk Scoring Model

최종 리스크 점수는 단일 금액으로만 계산하지 않는다.

다음 하위 점수를 0~100으로 정규화한 뒤 가중합으로 계산한다.

```text
Final Risk Score =
Financial Impact Score
+ Coverage Gap Score
+ Liquidity Stress Score
+ Recovery Difficulty Score
+ Public Risk Context Score
```

초기 가중치 후보:

```text
재무 충격             30%
준비 부족률           25%
유동성 압박           20%
회복 난이도           15%
공공통계 위험 맥락     10%
```

중요한 원칙:

```text
공공 발생률은 점수의 중심이 아니라 보정값으로 둔다.
```

이유:

- 이 프로젝트는 보험료 산정 모델이 아니다.
- 개인의 질병/사망/실직을 단정적으로 예측하지 않는다.
- 핵심은 발생 가능성보다 발생 시 감당 가능성이다.

## 10. Risk-Specific Analysis

### 10.1 Death Risk

핵심 질문:

```text
본인의 소득이 사라졌을 때 남은 가족이 필요한 돈은 얼마인가?
```

분석 요소:

- 부양가족 생활비
- 배우자 소득
- 자녀 수
- 자녀 나이
- 자녀 교육비
- 총 부채
- 주택담보대출 등 상환 필요 부채
- 장례 및 정리 비용
- 사망보험금 또는 기타 보장성 자산
- 연령/성별 사망률
- 주요 사망원인 통계

예상 필요금액 구조:

```text
사망 리스크 필요금액 =
가족 생활비 필요액
+ 자녀 교육비
+ 상환 필요 부채
+ 장례/정리 비용
- 배우자 또는 가족의 대응 가능 자산
- 사망 관련 보장금액
```

주의사항:

- 사망 리스크를 단순히 연소득 x 몇 년으로 계산하면 과대평가될 수 있다.
- 부양가족이 없는 사용자는 가족 생계 리스크보다 부채 정리, 장례비, 재무 정리 비용 중심으로 분석한다.

### 10.2 Critical Illness Risk

초기에는 중대질병 중 암을 중심 시나리오로 사용한다.

핵심 질문:

```text
질병으로 인해 치료비와 소득 공백이 발생하면 얼마가 부족한가?
```

분석 요소:

- 예상 치료비
- 입원비
- 수술비
- 비급여 가능 비용
- 간병비
- 치료 기간
- 소득공백 기간
- 질병 중 유지 가능한 소득
- 월 필수지출
- 월 부채상환액
- 진단비/실손/기타 보장금액
- 암 발생률
- 암 생존율
- 암 유병률

예상 필요금액 구조:

```text
중대질병 필요금액 =
예상 치료비
+ 간병비
+ 소득공백 기간 동안의 생활비
+ 소득공백 기간 동안의 부채상환액
- 질병 관련 대응 가능 자산
- 질병 관련 보장금액
```

주의사항:

- 암 발생률만으로 위험을 판단하지 않는다.
- 생존율과 치료 이후 회복기간도 재무 리스크에 중요하다.
- 건강보험 진료비 통계와 실제 본인부담액은 다를 수 있다.

### 10.3 Income Interruption Risk

핵심 질문:

```text
소득이 끊겼을 때 몇 개월 버틸 수 있고, 목표 기간 대비 얼마가 부족한가?
```

분석 요소:

- 고용 형태
- 직업
- 산업군
- 근속 연수
- 월 세후소득
- 배우자 소득
- 기타 소득
- 월 필수지출
- 월 선택지출
- 월 부채상환액
- 비상금
- 유동자산
- 실업급여 예상 여부
- 연령대별 실업률
- 산업별 고용 안정성
- 평균 재취업 기간

예상 필요금액 구조:

```text
소득중단 필요금액 =
목표 생존 개월 수 * (월 필수지출 + 월 부채상환액)
- 소득중단 중 예상 유입액
- 즉시 사용 가능한 유동자산
```

결과 예시:

```text
현재 생존 가능 기간: 2.1개월
목표 생존 기간: 6개월
부족 기간: 3.9개월
부족 금액: 15,600,000원
```

## 11. Input Design Direction

정밀 분석기이지만 입력이 너무 길어져서는 안 된다.

따라서 입력 단계를 3단계로 나눈다.

### Level 1. Basic Analysis

필수 입력:

- 나이
- 성별
- 결혼 여부
- 자녀 수
- 부양가족 여부
- 월 세후소득
- 월 지출
- 총 자산
- 총 부채
- 비상금

목적:

- 빠른 1차 리스크 분석
- 사용자가 서비스를 바로 이해할 수 있는 수준의 결과 제공

### Level 2. Detailed Analysis

정밀 입력:

- 배우자 소득
- 자녀 나이
- 부모 부양비
- 월 필수지출
- 월 선택지출
- 월 부채상환액
- 현금성 자산
- 단기 유동자산
- 투자자산

## 12. Asset and Protection Design

보험 정보는 상품 추천 단위로 다루지 않는다.

대신 리스크별 대응 가능 자원으로 해석한다.

예:

```text
사망보험금 -> 사망 리스크 대응 자원
암 진단비 -> 중대질병 리스크 대응 자원
실손보험 -> 의료비 리스크 대응 자원
소득보상성 보장 -> 소득중단/질병 리스크 대응 자원
```

피해야 할 표현:

```text
암보험이 부족합니다.
정기보험 가입이 필요합니다.
```

사용할 표현:

```text
중대질병 시나리오에서 예상 필요금액 대비 현재 확인된 대응 가능 금액이 부족합니다.
사망 시나리오에서 부양가족 생활비와 잔여 부채를 감당하기 위한 준비금이 부족합니다.
```

## 13. Result Report Direction

결과 리포트는 점수만 보여주지 않는다.

다음 항목을 함께 제공한다.

- 리스크 순위
- 최종 리스크 점수
- 리스크 등급
- 예상 필요금액
- 현재 대응 가능 금액
- 부족 금액
- 대응률
- 생존 가능 개월 수
- 회복 예상 기간
- 공공통계 기반 비교
- 계산에 사용된 주요 가정
- 사용자가 주의해야 할 해석상 한계

예시:

```text
1위: 소득중단 리스크

예상 필요금액: 24,000,000원
현재 대응 가능 금액: 8,000,000원
부족금액: 16,000,000원
생존 가능 기간: 2.1개월
목표 생존 기간: 6개월
회복 예상 기간: 18개월

분석 근거:
- 월 필수지출과 부채상환액을 기준으로 소득중단 시나리오를 계산했습니다.
- 동일 연령대 고용지표를 참고해 소득중단 리스크의 맥락 점수를 반영했습니다.
- 총자산이 아니라 즉시 사용 가능한 유동자산 중심으로 대응력을 평가했습니다.
```

## 14. Important Assumptions and Risks

설계 과정에서 반드시 주의해야 할 가정은 다음과 같다.

### 14.1 Total Assets Are Not Fully Available

총자산을 모두 대응 가능 금액으로 보면 안 된다.

부동산, 전세보증금, 퇴직연금, 장기 투자자산은 즉시 현금화하기 어렵다.

### 14.2 Insurance Benefits Must Be Risk-Specific

보험금은 리스크별로 분리해서 반영해야 한다.

예:

- 사망보험금은 실직 리스크 대응금이 아니다.
- 암 진단비는 사망 리스크 대응금이 아니다.
- 실손보험은 의료비 일부 대응 자원이지 소득공백 전체 대응금이 아니다.

### 14.3 Public Statistics Are Context, Not Prediction

공공 통계는 개인의 미래를 예측하는 근거가 아니다.

따라서 발생률은 최종 점수의 일부 보정값으로만 사용한다.

### 14.4 Average Values Can Be Misleading

소득, 자산, 부채는 평균값이 왜곡될 수 있다.

따라서 가능하면 평균보다 중위값, 분위값, 가구유형별 기준값을 우선 사용한다.

### 14.5 High Score Must Not Mean Insurance Recommendation

점수가 높다는 것은 보험 가입 필요가 아니라 재무 취약성이 높다는 뜻이다.

표현은 다음 방향을 유지한다.

- 대비 부족
- 현금흐름 취약
- 유동성 부족
- 회복기간 장기화
- 가족 생계 의존도 높음
