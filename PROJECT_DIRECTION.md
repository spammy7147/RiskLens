# RiskLens Project Direction

## 1. Positioning

RiskLens는 보험 상품 추천 서비스가 아니라, 개인의 생애 재무 리스크를 분석하는 서비스다.

핵심 관점:

- 보험을 추천하지 않는다.
- 특정 상품을 판매하지 않는다.
- 사용자의 재무 구조가 어떤 위험에 취약한지 계산한다.
- 위험 발생 시 예상 재무 충격과 부족금액을 보여준다.
- 공공데이터를 근거로 분석 방법과 해석 과정을 설명한다.

따라서 RiskLens의 목표는 단순 보험 계산기가 아니라, **공공데이터 기반 정밀 재무 리스크 분석기**를 만드는 것이다.

## 2. Project Goal

이 프로젝트는 빠르게 기능을 완성하는 것보다 다음 역량을 보여주는 것을 중요하게 둔다.

- 도메인 문제 정의
- 공공데이터 기반 분석 설계
- 재무 리스크 계산식 설계
- 설명 가능한 점수 모델 설계
- 백엔드 도메인 구조 설계
- 테스트 가능한 계산 로직 설계
- 분석 결과를 사용자에게 이해 가능한 리포트로 표현하는 능력

즉, 프로젝트의 목적은 “웹앱 하나 만들기”가 아니라, **분석 방법과 그 이유를 논리적으로 설명할 수 있는 포트폴리오 프로젝트**를 만드는 것이다.

## 3. Core Analysis Philosophy

RiskLens는 위험 발생 가능성을 예측하는 모델이 아니다.

중심 질문은 다음이다.

```text
위험이 발생할 가능성이 얼마나 높은가?
```

가 아니라,

```text
위험이 발생했을 때 현재 재무 구조로 감당 가능한가?
```

이다.

분석 구조:

```text
시나리오 기반 손실 계산
+ 개인 재무 대응력 분석
+ 공공통계 기반 위험 맥락
= 설명 가능한 재무 리스크 점수
```

## 4. Public Data Strategy

공공데이터는 사용자의 미래를 단정적으로 예측하는 용도가 아니다.

피해야 할 표현:

```text
당신은 암에 걸릴 확률이 높습니다.
당신은 사망 리스크가 높으므로 보험이 필요합니다.
```

사용할 표현:

```text
동일 연령대의 공공 통계상 질병/사망/고용 리스크를 참고했을 때,
이 사용자는 발생 가능성보다 발생 시 재무 충격과 회복 기간 측면에서 취약합니다.
```

공공데이터의 역할은 세 가지다.

| 역할 | 설명 |
| --- | --- |
| 기준선 | 동일 연령대, 성별, 가구유형, 소득구간과 비교 |
| 시나리오 근거 | 사망, 중대질병, 소득중단 시나리오의 기준값 설정 |
| 점수 보정 | 공공 통계상 상대적으로 중요한 위험을 최종 점수에 일부 반영 |

구체적인 출처와 지표 매핑은 `DATA_SOURCE_RISK_MODEL_MAPPING.md`에서 관리한다.

## 5. MVP Risk Scope

RiskLens가 장기적으로 다룰 수 있는 리스크는 많지만, 초기에는 다음 3개를 핵심 리스크로 둔다.

```text
1. 사망 리스크
2. 중대질병 리스크
3. 소득중단 리스크
```

다만 첫 구현은 3개를 동시에 얕게 만들지 않는다.

첫 번째 수직 슬라이스는 다음으로 고정한다.

```text
중대질병 발생 시 재무 충격 분석
```

이유:

- 질병별 의료비 공공데이터를 활용하기 좋다.
- 치료비, 소득공백, 생활비, 부채상환, 간병비를 함께 분석할 수 있다.
- Python 데이터 분석 학습 흐름을 만들기 좋다.
- 포트폴리오에서 데이터 분석 역량을 가장 잘 보여준다.

수직 슬라이스 상세 범위는 `RISK_ANALYSIS_VERTICAL_SLICE_DESIGN.md`에서 관리한다.

## 6. Risk Analysis Layers

전체 분석 모델은 4개 레이어로 나눈다.

| Layer | 역할 |
| --- | --- |
| Public Baseline Layer | 한국 공공통계 기반 기준값 |
| Personal Finance Layer | 사용자의 소득, 지출, 자산, 부채, 가족구성 |
| Scenario Simulation Layer | 사망, 중대질병, 소득중단 발생 시 재무 충격 계산 |
| Risk Scoring Layer | 손실규모, 부족률, 유동성, 회복기간, 공공위험맥락을 점수화 |

이 구조의 장점:

- 사용자 입력값만 계산하지 않는다.
- 공공 통계로 기준선을 만든다.
- 개인의 재무 구조를 기준선과 비교한다.
- 위험 발생 시나리오에서 부족금액과 회복기간을 계산한다.
- 최종 결과의 근거를 설명할 수 있다.

## 7. Core Metrics

RiskLens의 리포트는 점수 하나만 보여주지 않는다.

핵심 지표:

| 지표 | 의미 |
| --- | --- |
| Required Amount | 위험 발생 시 필요한 총 금액 |
| Available Amount | 현재 대응 가능한 금액 |
| Coverage Gap | 부족금액 |
| Coverage Ratio | 대응률 |
| Liquidity Survival Months | 유동자산으로 버틸 수 있는 기간 |
| Recovery Months | 부족금액을 회복하는 데 필요한 기간 |
| Risk Score | 위 지표들을 설명 가능한 방식으로 정규화한 점수 |

정확한 계산식은 `CALCULATION_SPEC.md`에서 관리한다.

## 8. Input Design Direction

정밀 분석기이지만 입력이 너무 길어져서는 안 된다.

입력은 3단계로 나눈다.

| 단계 | 목적 | 예시 입력 |
| --- | --- | --- |
| Level 1 Basic | 빠른 1차 분석 | 나이, 성별, 결혼 여부, 월소득, 월지출, 총자산, 총부채, 비상금 |
| Level 2 Detailed | 유동성/회복기간 분석 | 월 필수지출, 월 부채상환액, 현금성 자산, 자녀 나이, 배우자 소득 |
| Level 3 Advanced | 정밀 보정 | 고용형태, 산업군, 기존 보장금액, 소득 변동성, 건강 리스크 자기평가 |

MVP에서는 Level 1과 일부 Level 2만 사용한다.

## 9. Asset and Protection Principle

보험 정보는 상품 추천 단위로 다루지 않는다.

대신 리스크별 대응 가능 자원으로 해석한다.

| 입력 | 해석 |
| --- | --- |
| 사망보험금 | 사망 리스크 대응 자원 |
| 암 진단비 | 중대질병 리스크 대응 자원 |
| 실손보험 | 의료비 리스크 일부 대응 자원 |
| 소득보상성 보장 | 소득중단/질병 리스크 대응 자원 |

피해야 할 표현:

```text
암보험이 부족합니다.
정기보험 가입이 필요합니다.
```

사용할 표현:

```text
중대질병 시나리오에서 예상 필요금액 대비 현재 확인된 대응 가능 금액이 부족합니다.
```

## 10. Report Direction

결과 리포트는 다음 순서를 따른다.

1. 가장 취약한 리스크
2. 예상 필요금액
3. 현재 대응 가능 금액
4. 부족금액
5. 대응률
6. 회복 예상 기간
7. 점수와 하위 점수
8. 사용한 공공데이터와 가정
9. 해석상 주의사항

점수보다 원화 금액과 부족금액을 먼저 보여준다.

## 11. Important Assumptions

설계 과정에서 반드시 지켜야 할 가정은 다음과 같다.

### 11.1 Total Assets Are Not Fully Available

총자산을 모두 대응 가능 금액으로 보면 안 된다.

부동산, 전세보증금, 퇴직연금, 장기 투자자산은 즉시 현금화하기 어렵다.

### 11.2 Benefits Must Be Risk-Specific

보험금과 대응 자산은 리스크별로 분리해서 반영한다.

예:

- 사망보험금은 실직 리스크 대응금이 아니다.
- 암 진단비는 사망 리스크 대응금이 아니다.
- 실손보험은 의료비 일부 대응 자원이지 소득공백 전체 대응금이 아니다.

### 11.3 Public Statistics Are Context, Not Prediction

공공 통계는 개인의 미래를 예측하는 근거가 아니다.

발생률은 최종 점수의 일부 보정값으로만 사용한다.

### 11.4 Average Values Can Be Misleading

소득, 자산, 부채는 평균값이 왜곡될 수 있다.

가능하면 평균보다 중위값, 분위값, 가구유형별 기준값을 우선 사용한다.

### 11.5 High Score Must Not Mean Insurance Recommendation

점수가 높다는 것은 보험 가입 필요가 아니라 재무 취약성이 높다는 뜻이다.

표현은 다음 방향을 유지한다.

- 대비 부족
- 현금흐름 취약
- 유동성 부족
- 회복기간 장기화
- 가족 생계 의존도 높음
